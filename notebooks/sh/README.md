# 난임(불임) 시술 임신 성공 예측 — 최종 제출 정리

데이콘 대회: 난임 시술 임신 성공 여부 예측 (ROC-AUC)

## 최종 결과

| 항목 | 값 |
|---|---|
| **최종 Public LB Score** | **0.74187** |
| 채택 방법 | Seed Ensemble (5개 시드 평균) |
| 베이스 모델 | LightGBM + CatBoost + XGBoost + MLP (4종 힐클라이밍 블렌드) |

## 사용 방법

1. 캐글 노트북 환경에서 `stage_i_seed_ensemble_kaggle.ipynb`를 업로드
2. 데이콘 데이터셋(`train.csv`, `test.csv`, `sample_submission.csv`)을 Input으로 추가
3. 노트북 상단의 `DATA_DIR` 변수를 데이터 경로로 지정
   ```python
   DATA_DIR = "/kaggle/input/{본인 데이터셋 이름}"
   ```
4. GPU 가속기를 켜고 Run All 실행
5. 끝나면 `submission_seed_ensemble.csv`가 생성됨 → 이 파일을 제출

## 접근 과정 (요약)

전체 과정은 STAGE A부터 L까지, 단계적으로 가설을 세우고 검증하는 방식으로 진행했습니다.

### 1단계 — EDA 및 베이스라인 (STAGE A)
- train 256,351행 / test 90,067행, 69개 피처, 타깃 불균형(성공 25.8% vs 실패 74.2%)
- 핵심 발견: 나이와 성공률의 단조 감소 관계, 이식 배아 수의 비선형 임계 효과, DI 시술에서 배아/난자 관련 컬럼의 구조적 결측
- LightGBM 단일 베이스라인: OOF AUC 0.7395

### 2단계 — 피처 엔지니어링 가설 검증 (STAGE B/C)
- 연구 가설 5개(나이×난자출처 상호작용, ICSI 효과 반전, 배반포 단계 효과 등) 모두 multi-seed ablation으로 검증
- **결론: 전부 노이즈** — 트리 모델이 이미 원본 컬럼에서 해당 정보를 충분히 학습하고 있어 명시적 피처 추가 효과 없음

### 3단계 — 모델 다양성 (STAGE D, D-2)
- CatBoost에 원본 범주형 컬럼을 그대로 제공(정수 인코딩 대신), MLP 추가
- 로지스틱 회귀, fold-safe 타깃 인코딩, feature bagging 등으로 모델 간 상관 낮추기 시도
- 결과: 모델 간 Spearman 상관이 0.97~0.99로 여전히 높음, 다양성 추구의 효과는 미미(+0.0001 수준)
- **OOF 0.74020 / LB 0.74168**

### 4단계 — 하이퍼파라미터 튜닝 (STAGE E) ★ 가장 효과적인 단계
- Optuna로 LGB/CatBoost/XGBoost/MLP 4개 모델 각각 튜닝
- 적은 trial(FAST=True, 8/5/8/5)로도 의미 있는 개선 확인
- **OOF 0.74039 / LB 0.74187**
- trial을 60/40/60/40으로 늘린 본튜닝(FAST=False)은 OOF는 더 좋아졌지만(0.74059) LB는 오히려 하락(0.74182) — 과적합 신호로 판단

### 5단계 — 추가 시도들 (STAGE F~L, 대부분 효과 없음 또는 손해로 판정)
| 시도 | 핵심 내용 | 결과 |
|---|---|---|
| F. 스태킹 | 로지스틱/LightGBM 메타모델 | 힐클라이밍보다 항상 낮음 |
| G. 서브그룹 분리 | IVF/DI를 완전히 분리해서 모델링 | OOF 손해(-0.00062) |
| H. SHAP/Tree Interaction | 모델이 학습한 변수 상호작용을 명시적 피처화 | 노이즈(-0.00019) |
| J. CatBoost CTR | 교차 카테고리 통계 명시적 활용 | 거의 변화 없음(+0.00005) |
| K/L. 레이블 노이즈 점검 | 모델이 confident하게 틀린 행 제거 후 재학습 | **LB 하락(0.74134)** — 어려운 패턴 제거가 오히려 손해 |

### 6단계 — Seed Ensemble (STAGE I) ★ 최종 채택
- STAGE E(FAST=False)에서 찾은 best params를 고정
- 5개의 다른 시드(42, 7, 100, 777, 2024)로 5-fold 전체를 반복 학습 → 블렌드 예측을 평균
- 개별 시드 간 분산이 매우 작음(표준편차 0.00008) — 안정적인 학습 확인
- **OOF 0.74076 / LB 0.74187** (FAST=True 단일 시드와 동일한 최고 기록 유지, 분산 감소 확인)

## 핵심 교훈

1. **모델/블렌딩 다양성보다 하이퍼파라미터 튜닝이 더 효과적**이었음 (이 데이터 + 트리 모델 조합 한정)
2. **OOF 개선이 항상 LB 개선으로 이어지지 않음** — 더 정교한 탐색일수록 특정 fold 구성에 과적합될 위험 존재
3. 사전에 세운 가설 기반 피처 엔지니어링은 거의 전부 노이즈로 판정됨 — 트리 모델이 이미 원본 변수에서 상호작용을 충분히 학습
4. 레이블 노이즈로 보였던 행들은 실제로는 모델이 학습해야 할 진짜 어려운 패턴이었음 (제거 시 손해)
5. 다양한 방향(9가지 이상)을 체계적으로 시도한 결과, ROC-AUC 0.7419 부근이 현재 피처셋과 모델 조합에서의 현실적 상한으로 보임

## 데이터 누수 방지

전체 파이프라인에서 다음을 일관되게 지켰습니다:
- 모든 인코딩(라벨/원핫/타깃인코딩)과 스케일링은 **train fold 내부에서만 fit**, test/검증에는 적용만
- 5-fold 교차검증 구조를 모든 실험에서 동일하게 유지
- Adversarial Validation으로 train/test 분포 유사성 주기적 확인
